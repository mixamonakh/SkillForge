# SkillForge Learning System v2 — root package

**Репозиторий:** https://github.com/mixamonakh/skillforge  
**Назначение:** положить все файлы этого пакета в корень репозитория и запустить один Goal-запрос в Codex.

## Что находится в пакете

| Файл | Роль |
|---|---|
| `CODEX_GOAL_PROMPT.md` | Единственный текст, который нужно отправить в Codex Goal |
| `SFV2_IMPLEMENTATION_SPEC.md` | Полная продуктовая и техническая спецификация рефакторинга |
| `SFV2_TASK_LEDGER.md` | Полный последовательный backlog Phase 0–8; ни одна задача не должна быть потеряна |
| `SFV2_ADR_CAPABILITY_AND_AI.md` | Архитектурное решение по capability profile, адаптивности и AI |
| `SFV2_CONTRACTS_AND_SCHEMAS.md` | Предлагаемые TypeScript, JSON и доменные контракты |
| `SFV2_CONTENT_QUALITY_GATES.md` | Правила допуска контента от дизайна до production |
| `SFV2_AI_EVALUATOR_GOLD_SPEC.md` | Спецификация gold dataset для калибровки AI grading |
| `CHATGPT_CONTENT_GENERATION_PROMPT.md` | Отдельный промпт для генерации расширенного контента в браузерном ChatGPT |
| `OPENAI_API_SETUP_AFTER_REFACTOR.md` | Инструкция по оплате, ключу, `.env`, запуску и проверке AI после рефакторинга |

## Как использовать

1. Скопировать **все** файлы из пакета в корень `skillforge`.
2. Не удалять существующие `AGENTS.md`, `README.md`, `docs/`, `content/` и код.
3. В Codex включить режим **Goal**, модель/уровень рассуждения — максимально доступные для длительной разработки, разрешить агентов.
4. Вставить содержимое `CODEX_GOAL_PROMPT.md` одним сообщением.
5. Не передавать API-ключ во время рефакторинга. Codex обязан реализовать и протестировать AI через fake provider; реальный ключ подключается после завершения по `OPENAI_API_SETUP_AFTER_REFACTOR.md`.
6. После завершения рефакторинга открыть браузерный ChatGPT и использовать `CHATGPT_CONTENT_GENERATION_PROMPT.md` для создания следующих content packs.

## Почему документы лежат в корне

- Goal-агенту не требуется угадывать внешние пути или читать вложения из чата.
- Полная спецификация, contracts, quality gates и backlog доступны в одном checkout.
- `SFV2_TASK_LEDGER.md` служит устойчивой точкой продолжения, если Goal-сессия прервётся по техническому лимиту.
- Codex должен создать и обновлять `SFV2_EXECUTION_LOG.md`, поэтому повторный запуск с тем же prompt продолжит работу с первой незавершённой фазы.

## Что Codex имеет право изменить в этих файлах

- `SFV2_EXECUTION_LOG.md` создаётся и обновляется постоянно.
- Проектные документы можно уточнять, если актуальный код выявил конфликт или более безопасную реализацию.
- Нельзя молча удалять требования. Каждое изменение scope фиксируется в execution log с причиной.
- `CODEX_GOAL_PROMPT.md`, `CHATGPT_CONTENT_GENERATION_PROMPT.md` и эта инструкция не являются runtime-документацией приложения; после успешной реализации Codex может переместить их в `docs/archive/learning-system-v2/`, но только в самом конце и только если корневые ссылки останутся понятными.

## Граница ответственности

Этот пакет описывает рефакторинг и минимальный production-quality контент, необходимый для проверки нового цикла. Массовое расширение curriculum выполняется отдельно через browser prompt и проходит все quality gates. JSON в `content/packs/` остаётся canonical source.
