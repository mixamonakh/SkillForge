# ADR 0007: Без streak и давления регулярностью

- Статус: принято
- Дата: 2026-07-11

## Контекст

Цель SkillForge — доказуемая инженерная компетенция, а не частота открытия приложения. Streak, daily goal и сообщения о пропусках измеряют регулярность, создают чувство вины и могут стимулировать поверхностные действия вместо deliberate practice.

## Решение

Не хранить и не показывать streak, loss-of-series, daily heatmap/goal, punishment за pause и push «занятие пропущено». Пауза сама по себе не снижает mastery.

После configurable threshold (default 7 дней) показывать спокойный ResumeBanner с последней темой/позицией и `RETURN` session 15–20 минут: одно лёгкое retrieval и одно короткое применение. Review due является отдельным сигналом, а Dashboard показывает один следующий полезный шаг.

## Последствия

Положительные:

- продукт оптимизирует качество evidence, а не engagement vanity metric;
- возврат после жизненной паузы проще;
- пользователь не получает ложный негативный статус;
- UI соответствует взрослому инженерному тону.

Стоимость:

- нет привычного gamification hook;
- нельзя использовать streak как простую retention metric;
- recommendation/resume copy требует больше содержательного контекста.

Меры: evidence freshness, due reviews, session usefulness и load feedback дают продуктовые сигналы без давления. Никакая из этих метрик не отправляется third party по умолчанию.

## Рассмотренные варианты

- **Опциональный streak:** отклонён; даже opt-in меняет ценностный центр интерфейса и требует хранения/копирайта наказания.
- **Weekly goal без penalties:** не входит в MVP; возможен только после отдельного product decision и ADR, если не конкурирует с evidence.
- **Только скрыть streak, но хранить:** отклонено как ненужный privacy/engagement signal.
