# UX-принципы

## 1. Честность данных

До калибровки Dashboard показывает «SkillForge готов к калибровке», объясняет отсутствие процентов и предлагает начать JavaScript baseline. Не показываются readiness, список «слабых» тем или seeded динамика.

Неизвестное значение отображается как `—`, статус — «Нет данных»/`UNKNOWN`, а не `0%`/`WEAK`. Любая метрика сопровождается coverage и причиной достаточности данных.

## 2. Один следующий шаг

Dashboard отвечает «что сейчас?». После появления evidence он показывает одну primary recommendation, последнюю точку остановки, покрытие, до трёх review topics и последний import. Roadmap и Metrics могут быть плотнее, но не подменяют основной фокус.

## 3. Пауза без вины

Нет streak, daily goal, heatmap пропусков и сообщений об отставании. При паузе не менее настроенного threshold (по умолчанию 7 дней) показывается:

> С возвращением. Можно спокойно восстановить контекст за 15–20 минут.

Return-сессия содержит лёгкий retrieval и короткое применение. Пауза не понижает topic status; срок review хранится отдельно.

## 4. Сохранность ответа видима

Активные assessment/session показывают состояния `Сохраняется`, `Сохранено`, `Ошибка сохранения`. Draft сохраняется локально сразу и синхронизируется с API с debounce 700–1200 мс. При conflict пользователь получает server copy и понятное действие; несинхронизированная навигация защищена.

`Не знаю` — валидный ответ. Baseline не показывает подсказки и не раскрывает решение. Таймер может измерять duration, но не давит обратным отсчётом.

## 5. AI остаётся внешним аналитиком

Экспорт ясно отделяет локально проверенные результаты от свободных ответов. Импорт всегда проходит validation → preview → apply. Preview говорит, что исходные ответы останутся неизменными и показывает current/projected state после симуляции learning engine.

UI не обещает, что AI прав. Reliability и provenance видимы в деталях evidence.

## 6. Состояния каждого data screen

Каждый data-driven экран имеет:

- loading skeleton;
- empty state с осмысленным следующим действием;
- concise error и retry, requestId в деталях;
- insufficient-data state;
- populated state.

Длинные ошибки не прячутся в toast; raw stack в production не показывается.

## 7. Доступность

- WCAG 2.2 AA и видимый focus;
- все действия доступны клавиатурой;
- status передаётся текстом/иконкой, не одним цветом;
- labels и error descriptions связаны с полями;
- dialog управляет focus;
- у CodeMirror есть доступное описание;
- chart имеет текстовую summary/table;
- zoom 200%, mobile drawer, sticky session actions;
- `prefers-reduced-motion` отключает необязательные анимации;
- копирование текста не блокируется, звук не запускается автоматически.

## 8. Copy contract

Тон спокойный, инженерный и конкретный. Предпочтительно: «Недостаточно доказательств», «Тема пока нестабильна», «Готово к повторению». Запрещено: «Ты потерял серию», «Ты отстаёшь», «Вероятность пройти Яндекс — 73%», а также mastery-похвала после одной попытки.
